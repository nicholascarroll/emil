## Reporting a Vulnerability

We use GitHub's **Private Vulnerability Reporting**. 
Please navigate to the [Security tab](https://github.com/nicholascarroll/emil/security) and click "Report a vulnerability". 

Do not report security issues via public PRs or Issues.

### Issue Response Times

You will receive an initial response within 14 days.



## Hardening

Shell integration can be excluded from the build with the compiler flag `-DEMIL_DISABLE_SHELL`.


## Stable Releases

`emil` has not yet reached prerelease and is at a development version 0.1.0. 

There is a prerelease planned for early 2026 and a first release (version 1) in early 2027.
